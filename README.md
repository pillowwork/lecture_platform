## 데이터 설계
> 데이터는 크게 '강연 정보'와 '강연 신청 정보'로 나눈다.
> '강연 정보'에는 '강연장 정보'가 포함된다.
>* 강연 ``` LectureEntity ```
>* 강연장 ``` RoomEntity ```
>* 강연 신청 내역 ``` ApplyEntity ```

* 고민 / 설명
    * 고민
        * 많은 생각하지 말고, 주어진 시간 안에 심플하게 작성하자.
        * 엔티티간 관계 설정 잘못하면 성능 이슈, 관계는 약하고 가볍게 하는게 좋겠다.
        * 하지만, 그래도 강연에는 강연장 정보를 넣어야겠다.
    * 설명
        * BACK OFFICE API
        * FRONT API
